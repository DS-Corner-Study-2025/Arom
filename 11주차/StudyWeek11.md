## 12.1 웹 소켓 이해하기
- 웹 소켓
    - HTML5에 새로 추가 된 실시간 양방향 데이터 전송을 위한 기술
    - 최신 브라우저 대부분 지원
        - 노드의 경우 ws나 Socket.IO를 통해 이용

-  Socket.IO 
    - 웹 소켓을 활용한 라이브러리


- 폴링(polling) 
    - 웹 소켓 등장 이전의 실시간 데이터 전송 방식
    - HTTP 기반 구현
        - HTTP : 클라이언트 -> 서버 단방향 통신 
        -  주기적으로 서버의 새 업데이트 검사 -> 변경점 가져오기

- 웹 소켓과 폴링
    - 폴링 : 서버의 지속적 검사 필요
    - 웹 소켓 
        - 이벤트 기반 작동 
        - 웹 소켓 연결 이후, 업데이트가 있는지 요청을 보내 검사할 필요 없음 
        -  웹 브라우저와 웹 서버가 계속 연결된 상태로 존재
        - 업데이트 내용 발생 -> 즉시 서버에서 클라이언트에게 알림
        - HTTP 프로토콜과 포트 공유 가능
        - 폴링방식에 비해 성능 개선

- 서버센트 이벤트(Server Sent Event)
    - Event Source 객체 이용
    - 연결 이후, 서버가 클라이언트에 지속적으로 데이터 전송
    - **웹소켓과의 차이 : 서버에서 클라이언트 단방향 통신
    - 굳이 양방향 통신을 할 필요 없는 기능은 SSE로 구현
